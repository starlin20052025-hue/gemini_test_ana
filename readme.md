## 二、目前我在意的核心要求

### 1. 歷史資料要全抓
不要只抓近幾年，也不要只抓局部資料。  
我要你假設：**能抓多少就抓多少，盡量從最早可取得時間一路抓到最新。**

我要你評估：
- 為什麼「全歷史回測」重要
- 全歷史回測與只測局部期間，會對 CAGR / MDD / Sharpe / Calmar 造成什麼偏差
- 如果市場 regime 不同（牛市 / 熊市 / 盤整），全歷史有什麼好處與壞處

---

### 2. 必須分訓練集與測試集
這是硬需求。  
我不接受「全資料一起回測後挑最好的參數」這種方式。

我要你幫我設計合理方案，例如：
- train / validation / test
- anchored walk-forward
- rolling window
- expanding window
- nested validation（如果你認為有必要）

請你明確分析每種方案的：
- 優點
- 缺點
- 適合哪種策略
- 實作複雜度
- 對加密貨幣這種非平穩市場的適配性

最後請你**給出你最推薦的一種方案**，不是只列一堆選項。

---

### 3. 嚴格避免未來資訊滲漏（future leak）
我很重視這件事。  
請你從以下角度檢查：

- 指標計算是否偷看到未來
- 標準化 / normalization 是否偷看到未來
- 參數搜尋是否用了測試集結果
- walk-forward 是否真的只用過去資料
- bar close 後訊號、下單時點、持倉 shift 是否正確
- 若使用 K 線與技術指標，哪些寫法最容易不小心 leak

我要你列出：
1. 常見錯誤清單
2. 正確做法
3. 你建議我在程式架構中加入哪些防呆機制

---

### 4. 週期選擇：1h、4h、1d
我目前在思考週期問題。  
有人預設用 1h，但我不想只是「習慣上用 1h」這種理由。

請你分析：
- 1h 的優點與缺點
- 4h 的優點與缺點
- 1d 的優點與缺點
- 不同週期對於：
  - 雜訊
  - 訊號頻率
  - 手續費敏感度
  - slippage 敏感度
  - regime 切換偵測
  - 訓練樣本數
  - 過擬合風險
  - live trading 可執行性
  會有什麼影響

最後請你不要模糊帶過，直接回答：
- 如果是第一版 baseline research framework，你建議先用哪個週期，為什麼
- 第二階段擴充時，應該怎麼引入多週期比較

---

### 5. 我需要的績效指標
請你明確告訴我，最少應該輸出哪些績效數據。  
至少要包含：

- Sortino
- Win rate
- Profit factor
- Exposure
- Trade count
- Avg holding period
- Recovery factor
- Ulcer index
- Turnover

請說明每個指標的意義，以及哪些指標容易誤導。
